# axum-with-custom-extractors

Like [`axum-hello-world`](../axum-hello-world), but with custom `JsonV` and `QueryV` extractors that validate request data automatically. Handlers never call `.validate()` themselves -- invalid requests are rejected before the handler runs.

## How it works

`JsonV<T>` and `QueryV<T>` are thin wrappers around axum's `Json<T>` and `Query<T>`. They require `T: Validation` and call `.validate()` after deserializing. If validation fails, the extractor returns a structured JSON error response (422 Unprocessable Entity) without the handler being invoked.

```rust
// before: manual validation in every handler
async fn create_greeting(
    Json(body): Json<CreateGreetingRequest>,
) -> Result<(StatusCode, Json<Greeting>), (StatusCode, String)> {
    if let Err(e) = body.validate() {
        return Err((StatusCode::BAD_REQUEST, e.to_string()));
    }
    // ...
}

// after: validation happens in the extractor
async fn create_greeting(
    JsonV(body): JsonV<CreateGreetingRequest>,
) -> Result<(StatusCode, Json<Greeting>), AppError> {
    // body is already validated
    // ...
}
```

See [`hello-world/src/extract.rs`](hello-world/src/extract.rs) for the full implementation.

## Structure

```
.
├── Cargo.toml                  # workspace root
├── openapi.yaml                # API spec (source of truth)
├── hello-world/                # your application
│   └── src/
│       ├── extract.rs          # JsonV, QueryV, AppError
│       └── main.rs             # axum handlers (no manual validation)
└── hello-world-openapi/        # generated by openapi-modelgen
    └── src/
        ├── model.rs            # Greeting, CreateGreetingRequest, ListGreetingsQuery, ...
        └── validation.rs       # Validation trait impls
```

## Regenerating the types

From the openapi-modelgen project root:

```sh
cargo run -- \
  --input examples/axum-with-custom-extractors/openapi.yaml \
  --output-dir examples/axum-with-custom-extractors \
  --crate-name hello-world-openapi \
  --workspace
```

## Running

```sh
cd examples/axum-with-custom-extractors
cargo run -p hello-world

# In another terminal:
curl http://127.0.0.1:3000/greetings
curl http://127.0.0.1:3000/greetings?language=de&limit=2

# Create a greeting
curl -X POST http://127.0.0.1:3000/greetings \
  -H 'Content-Type: application/json' \
  -d '{"message": "Ciao, mondo!", "language": "en", "tags": ["casual"]}'

# Validation error (empty message) -- returns 422 with structured JSON
curl -X POST http://127.0.0.1:3000/greetings \
  -H 'Content-Type: application/json' \
  -d '{"message": "", "language": "en"}'
```
